Diagnóstico da série¶

Primeiramente, tomamos a srie de retornos, por ser estacionria e ergdica.

In [ ]:
df$ret <- (df$Close - lag(df$Close)) / lag(df$Close)
df$ret[1] <- 0
retplot <- ggplot(df, aes(x = Date, y = ret)) +
    geom_line() +
    labs(x = "Data", y = "Retorno", title = "Retorno diario do Google")

retplot

A srie de retornos possui picos e vales de diferentes tamanhos. Isso indica heteroscedasticidade da srie, ou seja, a varincia no constante. Podemos utilizar o teste dos multiplicadores de Lagrange para refutar ou confirmar essa hiptese.

In [ ]:
library(dynlm)
library(FinTS)

# Modela a mdia
df.mean <- dynlm(df$ret ~ 1)
# Resduos quadrados
ehatsq <- ts(resid(df.mean)^2)
# Regresso em resduos quadrados defasados
df.ARCH <- dynlm(ehatsq ~ L(ehatsq))

df.archTest <- ArchTest(df$ret, lags = 1, demean = TRUE)
df.archTest

O valor-p do teste $< 0.05$, portanto rejeita-se a hiptese nula de que a srie no possui efeitos de um ARCH.

Transformar uma srie em uma srie de retornos torna seus pontos so no-correlacionados, restando somente a varincia, o que torna-as adequadas para uso em um modelo GARCH. Podemos comprovar este fato observando que o autocorrelograma da srie no possui valores elevados.

library(rugarch)

# Funo que ajusta um modelo GARCH com os parmetros passados
fit_garch <- function(
        x,
        ar_lag, 
        ma_lag, 
        arch_lag, 
        garch_lag, 
        model_type, 
        distribution_type
) {
    spec <- ugarchspec(variance.model = list(model = model_type, garchOrder = c(arch_lag, garch_lag)),
                       mean.model = list(armaOrder = c(ar_lag, ma_lag)), 
                       distribution.model = distribution_type
    )

    model_name <- paste0('ARMA(', ar_lag, ',', ma_lag, ')+', model_type, 
        '(', arch_lag, ',', garch_lag, ') ', distribution_type)
    # message('Estimando modelo ', model_name, '...')
    
    try ({
        fit <- ugarchfit(spec = spec, data = x)
    })
    if (!is.null(coef(fit))) {
        AIC <- rugarch::infocriteria(fit)[1]
        BIC <- rugarch::infocriteria(fit)[2]
    } else {
        # Estimativa falhou
        AIC <- NA
        BIC <- NA
    }

    est_tab <- tibble(ar_lag, 
                      ma_lag,
                      arch_lag,
                      garch_lag,
                      AIC =  AIC,
                      BIC = BIC,
                      model_type = model_type,
                      distribution_type,
                      model_name = model_name) 
    return(est_tab)
}

simulate <- function(data, n_sim, n_days_ahead, model) {

    single_sim <- function(i_sim, n_days_ahead, model) {
        rugarch_sim = ugarchsim(model, n.sim = n_days_ahead, m.sim = 1)
        sim_series <- rugarch_sim@simulation$seriesSim
        sim <- tibble(
            i_sim = i_sim,  # Id da simulao
            i_t = 0:length(sim_series),  # Index da previso
            date = last(data$Date) + i_t,  # Data da previso
            sim_ret = c(0, sim_series),  # Retorno simulado
            sim_price = last(data$Close) * (cumprod(1 + sim_ret))  # Preo simulado
        )
        return(sim) 
    }

    # Realiza as simulaes
    out <- bind_rows(
        map(.x = 1:n_sim, 
            .f = single_sim, 
            n_days_ahead = n_days_ahead,
            model = model
        )
    )

    return(out)
}

Conclusão¶

Nenhum modelo capaz de considerar todas as imprevisibilidades do mercado financeiro, e portanto este modelo limitado em sua capacidade de previso, ainda mais para um perodo de trs anos, em que diversos eventos podem mudar a tendncia do valor das aes (como quando a pandemia de COVID-19 visivelmente influenciou no crescimento das aes entre 2020 e 2021).

Tendo isso em mente, podemos colocar um bom grau de confiana em uma informao que o resultado deste modelo nos providencia: no cenrio atual, existe uma boa chance de que as aes da Google retornaro ao seu pico de 2021 entre um ou dois anos a partir de hoje.
